NAME

Metamod::Config - get runtime configuration environment

SYNOPSIS

  use Metamod::Config;

  if(!Metamod::Config->config_found($config_file_or_dir)){
    die "Could not find the configuration on the commandline or the in the environment\n";
  }

  my $conf = Metamod::Config->new($config_file_or_dir);
  my $var = $conf->get($varname);

DESCRIPTION

This module must be used to read the configuration file. It is initialised during startup of MetamodWeb.

FUNCTIONS

Metamod::Config->new([configfilename], [options])

is this up to date? FIXME)

Initialize the configuration with a config-file. If no config-file is given, the environment-variable METAMOD_MASTER_CONFIG will be used (useful for testing), otherwise, the default config-file located in '../../master_config.txt' relative to the installation of Metamod::Config will be used.

This function will die if the config-file cannot be found.

This function makes sure, that each config-file will only be opened once, even if the same config file is opened several times.

Options

nolog

Skip logging initialization

Metamod::Config->instance()

Get the current config instance (which MUST be set with new() previously).

Metamod::Config->_reset_singleton()

Undefs the current singleton object. The ONLY reason to use this is for testing of the class itself.

Metamod::Config->config_found($file_or_dir)

Check if the class can find a config either in the supplied parameter or the enviroment. Dies if the found config does not actually exist.

$file_or_dir

A variable (possibly empty) with the path to the config file or directory.

return

Returns the path to the config file. If $file_or_dir is a file or directory the parameter is returned. If that variable is empty the value of METAMOD_MASTER_CONFIG is returned.

$conf->set($varname, $value)

Set a configuration variable

$conf->unset()

Unset a configuration variable

$conf->get($varname)

return the configuration variable configVar as currently set. This will reread the config-file each time it has been changed. Gives a warning if not specified in master_config (mostly for historical reasons before sensible defaults were implemented).

$conf->getall()

returns all configuration variables as a hash

$conf->getallflags()

returns all configuration flags as a hash. Each flag is a bitfield composed of:

    bitmask:    description:
        1       default value set
        2       master value set
        4       envvar value set
        128     warning flag

$conf->has($varname)

return true if the configuration variable configVar is currently set. This will reread the config-file each time it has been changed. Does not give any warnings.

$conf->is($varname)

return true if the configuration variable has been set and is not among a list of false values (0, false, no, off, empty string). Does not give any warnings.

$conf->split($varname)

Splits a config variable separated by newlines, commas or spaces, optionally delimited by quotes (single or double – quotes are mandatory for comma separated lists).

If multi-line table, returns a hash of key/value pairs, where value is either a string or a list of strings. If single-line, returns a list, or if only one element, a string.

$conf->getDSN()

Return the database source name of the metadata database, i.e. "dbi:Pg:dbname=damocles;host=localhost;port=5432"

$conf->getDSN_Userbase()

Return the database source name of the user-database, i.e. "dbi:Pg:dbname=userbase;host=localhost;port=5432"

$conf->getDBH()

Return a cached/pooled DBI-handler to the default database of metamod. The handler is in AutoCommit = 0 and RaiseError = 1, FetchHash mode. This function will die on error. (DBI-connect error)

disconnect will free the database. Be careful when using getDBH and transactions. A call to getDBH will commit a transaction, and cached connections might be used several places.

$conf->initLogger()

Initialise a Log::Log4perl logger.

$conf->config_dir()

Get the directory that the master config file is located in. Can be used to get paths relative to the config file.

return

The path to the directory containing the configuration file.

$conf->installation_dir()

Get the absolute path to the directory where the METAMOD application is installed.

return

Returns the absolute path to the directory where METAMOD is installed. Throws an exception if the location cannot be determined.

$conf->getVarNames()

return

A list of all the variables names in the configuration file.

$conf->getDSFilePath($ds_name)

Get the file path to where the XML metadata files are stored on disk.

This method will die if one of the directories on the path to the file is not readable.

$ds_name

The name of the dataset on the form '<application>/<dataset name>' or '<application>/<parent name>/<dataset name>'

return

The path to the basename of the XML metadata files. I.e. the .xml or .xmd ending is not part of it.

$conf->path_to_config_file($filename, @dirnames)

Get the path to a file that is either located in the application configuration catalog or in the config/ catalog.

Parameters

$filename

The name of the file.

@dirnames

A list of directory names that gives the location of the file relative to the application config catalog or the config/ catalog.

return

The path to the file in the application configuration catalog if it exists. Otherwise it returns the path to the file in the config/ catalog.

Dies if the file cannot be found any of the places.

$conf->external_ts_plot()

Return URL to external timeseries plotter, or undef if not enabled

Parameters

$opendap

URL to OPeNDAP service for dataset

$vars

Comma separated list of variables to plot

$conf->version()

Return the METAMOD version number and build date
