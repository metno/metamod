NAME

MetamodWeb::Utils::Questionnaire - Utility functions for the questionnaire.

SYNOPSIS

DESCRIPTION

FUNCTIONS/METHODS

$self->quest_validator($config_file)

Generate a MetamodWeb::Utils::FormValidator form profile from the questionnaire configuration file.

$config_file

The full path the to questionnaire configuration file in JSON format.

return

Returns a hash reference with the form profile that can be used for validation.

$self->quest_config($config_file)

Get the configuration of a questionnaire from a config file.

$config_file

The full path to the configuration file. The configuration file is exected to be a JSON file where consisting of an array of hashes. Each element in the array describes one UI element in the questionnaire.

return

Returns the configuration as reference to list of hash references.

$self->load_anon_metadata($config_id, $response_key)

Load metadata (and info) from a 'anonmous' dataset. That is a dataset that is not stored in the userbase.

$config_id

The id of the questionnaire that the metadata is related to.

$response_key

The key that the user will use to identify the response.

return

Returns the metadata for the dataset as a hash reference.

$self->save_anon_metadata($config_id, $response_key, $metadata)

Save metadata to an anonomous dataset. That is a dataset that is not stored in the userbase.

$config_id

The id of the questionnaire that the metadata is related to.

$response_key

The key that the user will use to identify the response.

$metadata

A hash reference of the metadata to store. The values in that has should be array references, even for single values.

return

Returns true if the metadata was save successfully. Dies on error.

$self->load_dataset_metadata($userbase_ds_id)

Load the metadata for a dataset that is stored in the userbase.

$userbase_ds_id

The ds_id from the userbase.

return

Returns the metadata as a hash reference.

$self->save_dataset_metadata($config_id, $userbase_ds_id, $metadata)

Save metadata to a dataset that is stored in the userbase.

$config_id

The id of the questionnaire that the metadata is related to.

$userbase_ds_id

The ds_id from the userbase for the dataset.

$metadata

The metadata as a hash reference. The values should be array references even for single values.

return

$self->_save_metadata($metadata, $dataset_path, $ownertag )

Helper method for storing metadata and info for a dataset.

$metadata

The metadata to store as a hash reference.

$dataset_path

The path on disk where the metadata should be stored.

$ownertag

The ownertag that should be used for metadata.

$metabase_ds_name (optional)

The ds name as found in the metabase. This is only relevant for datasets and not anonymous metadata not tied to a specific dataset.

return

$self->_load_metadata($dataset_path)

Helper method for loading the metadata for a dataset.

$dataset_path

The location on disk where the dataset is stored. This can refere to a location that does not yet exist in the case of a new dataset.

return

The metadata as a hash reference.

$self->load_dataset($dataset_path)

Load the dataset as a Metamod::Dataset object. In the case of the dataset not yet existing a new empty dataset will be created.

$dataset_path

The path on disk where the XML metadata files are stored.

return

Returns a Metamod::Dataset object.

$self->dataset_path($userbase_ds_id)

$userbase_ds_id

The ds_id of the dataset from the userbase.

return

Returns the path to where the XML metadata files are stored on disk.
