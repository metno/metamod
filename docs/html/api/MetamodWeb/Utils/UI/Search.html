NAME

MetamodWeb::Utils::UI::Search- Utility functions for building the search ui.

SYNOPSIS

DESCRIPTION

FUNCTIONS/METHODS

$self->search_categories()

Get a list of all the search categories for this instance.

return

Returns a reference to a list of hash references. Each hash reference has the following keys: sc_id, sc_idname, sc_type. In addtion it the parsed values from sc_fnc are included as key/value pairs.

$self->selected_bks($category_id)

Get the selected basic keys for a category.

$category_id

The sc_id of the category.

return

Returns a hash reference with all the selected basic keys for the category. The key is the basic key id and the value is the name of the basic key.

$self->basickeys($category_id)

Get all the basic keys for a category.

$category_id

The sc_id for the category.

return

Returns the basic keys as reference to a list. Each element in the list is hash reference with the colum values for the basic key.

$self->two_way_table_h_keys($two_way_table)

$two_way_table

A two way table structure as returned by Metamod::DBIxSchema::Metabase::ResultSet::Dataset::two_way_table.

return

Returns all the horisontal keys in the two way table sorted alphabetically.

$self->num_search_cols()

return

Returns SEARCH_APP_MAX_COLUMNS from master_config.txt

$self->search_options()

Get the search options for which metadata to use in the search result.

return

A reference to a list of hash references.

$self->default_mt_name($col_num)

$col_num

The column number to get the default mt_name for.

return

The mt_name of the default column for the specific $col_num.

$self->default_vertical_mt_name()

return

The default mt_name for the vertical column in a two way table.

$self->default_horisontal_mt_name()

return

The default mt_name for the horisontal column in a two way table.

$self->search_app_show_columns()

This function parses the SEARCH_APP_SHOW_COLMNS configuration variable from the master config and returns the result as a list of hash references.

return

Returns an array reference with hash references. Each hash reference contains information about one metadata type that can be displayed in the search result.

$self->mt_name_to_display_name($mt_name)

$mt_name

A <mt_name> column value.

return

Returns the display name for a metadata column if it exists in SEARCH_APP_SHOW_COLUMNS. Otherwise just returns the mt_name again.

$self->selected_map()

return

Returns the SRID for the currently selected map. If the user has not selected a map yet, the first map in the map list will be selected.

$self->available_maps() [DEPRECATED]

return

Returns a reference to a list of available maps. Each map is returned as a hash reference with the following keys: srid, name and optionally <selected> for the currently selected map.

map_coordinates()

return

Returns a hash reference with the currently selected map coordinates. The map coordinates are given as two points. The keys in the hash are: x1, y1, x2 and y2.

$self->topics_tree()

Generate the hierarchical topics and variables tree.

return

Returns a reference to a list of roots. Each root is the start of a tree structure.

$self->selected_topics()

Get a list of all the selected topics.

return

Returns a reference to a list of selected topics. Each selected topic is a hash reference with the following keys: type, name and id

$self->level2_result( $dataset )

Get the datasets at level 2 that related to the supplied dataset at level 1. The level 2 datasets will be paged across several pages if there are several results.

$dataset

A DBIx::Class row object for dataset. It is expected that this dataset is at level 1.

return

A DBIx::Class resultset for the level 2 datasets. The dataset will be paged.

level2_metadata_columns($dataset)

Calculate which metadata columns to display at level 2 in the search result for a particular dataset.

A metadata column should be displayed at level 2 if the metadata column has data and it is different from the value at level 1.

$dataset

A DBIx::Class row object for the level 1 dataset that the level 2 datasets belong to.

return

A reference to a list of metadata column names.

$self->looks_like_url($dataref)

Check if a dataref looks like an URL so that we can link to it.

$dataref

The dataref string to check.

return

Returns true if we can link to it. False otherwise.

$self->remove_hidden_flag($basickeys)

Removes the textual hidden flag that is present on basic keys names that are associated with hierarchical keys.

$basickeys

An array ref with basic keys.

return

An array ref where the '> HIDDEN' flag has been removed from any basic key where it is present.

$self->urls_to_links($md_content)

Replace text URLs in the content with HTML links.

$md_content

A reference to a list of md_content values.

return

A reference to a list of md_content values where the URLs have been replaced with HTML links.

$self->dataset_in_userbase($ds)

Check wether a level 1 dataset is found in the user base or not.

$ds

A DBIx::Class row object for the dataset in the metabase.

return

Returns 1 if dataset is also found in the user base. False otherwise.

$self->pages_to_show($pager, $max_range)

Calculate the list of page numbers to show in the pager navigation. The pages to show will be between current page - $max_range and current page + $max_range. First and last page constraints will be respected.

$pager

A Data::Page object.

$max_range (default = 10)
return

An array reference with the page numbers to show.

$self->navigation_url($url_template, $page)

IMPLEMENTATION REMARK: This funciton is only needed since Template::Toolkit does not have a sprintf function.

$url_template

A printf compatible string with a single %s parameter

$page

The page that should be placed in the $url_template

return

The $url_template string with the $page inserted at the correct place.
